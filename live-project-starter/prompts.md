# Live Project Prompts

Students can copy these prompts into opencode, Claude Code, or Codex while creating their own dashboard project.

## 1. Inspect The Brief

```text
Read docs/live-project.md and summarize the project requirements. Do not create files yet.
```

## 2. Choose A Dataset

```text
Help me choose a beginner-friendly Kaggle dataset for a small dashboard. The dataset should have a CSV file, clear columns, and enough categories or dates to make at least three charts. Do not download anything or create files yet.
```

## 3. Create The Files

```text
Create a new student-dashboard/ folder with app.py, requirements.txt, .gitignore, and a short README.md. Use Python, pandas, Streamlit, and Plotly. Assume I will put one Kaggle CSV file in student-dashboard/data/. Do not include a dataset or credentials.
```

## 4. Inspect The Data

```text
Inspect the CSV in student-dashboard/data/. Explain the columns, missing values, and three chart ideas before editing app.py.
```

## 5. Build Version One

```text
Build the first version of the dashboard. Include a title, dataset summary, at least three charts, one filter or selector, and a short notes section. Keep app.py small enough for beginners to read.
```

## 6. Verify Manually

```text
Tell me how to run the dashboard with Streamlit and what to check in the browser. Do not add new files.
```

## 7. Review

```text
Review the dashboard for chart clarity, beginner-readable code, missing-data handling, and unnecessary complexity. Do not edit files until I approve the changes.
```

## 8. Improve One Thing

```text
Apply one small improvement from the review. Keep the diff small and explain exactly what changed.
```
