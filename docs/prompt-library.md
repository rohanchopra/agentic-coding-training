# Prompt Library

These prompts are intentionally simple. The workflow matters more than clever prompt wording.

## Inspect First

```text
Inspect this repository and explain the main files. Do not edit files.
```

## Ask For A Small Edit

```text
Make one small wording improvement. Keep the meaning the same. Summarize exactly what changed.
```

## Create A Dashboard Starter

```text
Create a student-dashboard/ starter with app.py, requirements.txt, .gitignore, and README.md. Use Python, pandas, Streamlit, and Plotly. Do not include a Kaggle dataset or credentials.
```

## Review Changes

```text
Review the current changes. Tell me what changed, what looks risky, and what I should check manually. Do not edit files.
```

## Debug A Simple Problem

```text
The dashboard does not look right in the browser. Inspect app.py, explain the likely problem, and suggest the smallest fix before editing.
```

## Inspect A Dataset

```text
Inspect the CSV in student-dashboard/data/. Explain the columns, missing values, and three possible charts. Do not edit files yet.
```

## Explain For Students

```text
Explain what just changed in beginner-friendly language. Include what the human asked for and what the tool changed.
```

## Compare Tools

```text
Compare how this same beginner task would feel in opencode, Claude Code, and Codex. Focus on practical differences, not marketing claims.
```
